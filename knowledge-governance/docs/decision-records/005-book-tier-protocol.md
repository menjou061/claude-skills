# ADR-005 書籍分級閱讀協議（Book-Tier Reading Protocol）

## Status
Accepted (2026-05-31)

## Context

CZ《幣安人生》測試暴露了一個系統性風險：AI 替用戶讀書、用戶內化不足。
- 用戶未親讀原文時，AI 提取的規則候選缺乏個人化內化
- 規則晉升的「Gate 6 個人化回測」依賴用戶對原文的真實理解，否則回測退化為對 AI 摘要的二次認同
- 一旦 approve 的規則來自未親讀的書，規則庫質量會隨時間稀釋

核心矛盾：
- **效率角度**：AI 可以快速處理任何書
- **質量角度**：規則庫的有效性依賴用戶對源頭的真實理解
- **解法**：按書的影響等級分級，匹配對應的閱讀深度要求

## Decision

書籍按 Tier 分級，匹配閱讀深度與處理流程：

### Tier A：規則級書（影響投資/工作核心決策）
- **用戶必讀**：完整閱讀原文，不允許跳過
- **AI 角色**：在用戶讀完後提取候選，作為「盲點 mirror」（你可能漏掉的視角）
- **示例**：CZ《幣安人生》、Ray Dalio《原則》、Howard Marks《投資最重要的事》

### Tier B：參考類書
- **用戶**：可選擇性閱讀
- **AI 角色**：提取候選後，**用戶必回讀對應段落**，才能進入 Gate 5/6
- **示例**：行業報告、技術文檔、方法論彙編

### Tier C：消遣類書
- **用戶**：自由閱讀
- **AI 角色**：摘要不入規則候選池
- **示例**：傳記消遣、小說、隨筆

### 子場景處理

| 場景 | 用戶狀態 | 處理 |
|------|---------|------|
| A.0 | 完全未讀 Tier A | **暫停採集**，先讀完再來 |
| A.1 | 完全讀完 Tier A | 正常 7 閘審查 |
| A.2 | 部分讀（如 50%）Tier A | AI 提取後，用戶必補讀對應段落後再 Gate 5/6 拍板 |
| B.x | 任何 Tier B | AI 提取 → 用戶回讀段落 → Gate 5/6 |
| C.x | 任何 Tier C | AI 摘要僅入 wiki/資料摘要/，不入候選 |

## Consequences

**正面：**
- 規則庫每條規則背後都有用戶親讀的原文支撐
- 避免 AI 摘要替代用戶理解，降低規則庫稀釋風險
- Tier A 書的處理時間延長，但規則質量大幅提高

**負面：**
- 採集吞吐量下降（Tier A 需用戶完整閱讀）
- 增加用戶判斷成本（每本書先定 Tier）

**緩解：**
- AI 在採集入口主動詢問 Tier 標記
- Tier C 仍可生成 wiki 摘要（用於資料檢索），只是不入規則候選

## Constraints（紅線）

- **任何 approve 規則必須親自讀過對應原文段落**
- AI 不得在 Tier A.0 場景下強行提取候選
- AI 在 Tier A.2 / B 場景下必須引用具體章節，供用戶回讀

## Implementation

1. 採集入口（book-digest skill）加入 `[tier=A|B|C]` 標記
2. 若用戶未指定，AI 主動詢問：「這本書屬於 Tier A 規則級 / B 參考類 / C 消遣類？」
3. Tier A 進入採集前，AI 確認：「您是否已完整讀完此書？（完全讀完 / 部分讀完 / 完全未讀）」
4. Tier A.2 / B：AI 在 candidates 中標注 `回讀段落: [章節名]`，Gate 5 前用戶必須確認已回讀
5. 配額仍受 ADR-004 限制（一本書晉升 3-7 條）
