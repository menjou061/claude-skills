# knowledge-governance — 個人決策操作系統

## 1. 項目名稱

knowledge-governance — 個人決策操作系統。以 Obsidian 為真源、Claude Code 為處理器，將個人知識管理從「收藏夾」升級為可驅動決策的閉環系統。

---

## 2. 問題陳述

每天從多源（書/播客/文章/會議/市場/對話）採集信息，但散落、無結構、無法回饋到具體決策（交易/產品/個人能力），導致同樣錯誤反復發生、好洞察被遺忘、決策效率無法持續提升。核心矛盾不是信息量不足，而是採集與決策之間缺乏可追溯的結構化橋樑。現有工具（Obsidian 筆記、交易日志）各自孤立，尚未形成跨域的統一閉環。

---

## 3. 用戶

單用戶（owner），本地使用，不涉及多人協作或雲端服務化。Obsidian 為唯一真源，Claude Code 作為處理器執行採集、結構化、策展與反饋任務。用戶具備較強的系統設計意識，傾向於「人在環路」的半自動化模式，而非全自動黑盒。

---

## 4. 核心價值主張

構建可擴展、可迭代、可維護的個人系統，實現完整閉環：

- **採集標準化**：任意輸入源走統一流水線，輸出格式一致的結構化卡片
- **處理結構化**：卡片可被檢索、可被引用、可被版本追蹤
- **決策可追溯**：重要決策（交易/產品/個人能力）能追溯依賴的卡片來源
- **反饋可回灌**：決策結果反向優化規則和 schema，系統隨使用持續進化

---

## 5. 系統觀（五層）

| 層級 | 說明 | 現狀 |
|------|------|------|
| **採集層** | books / podcasts / articles / meetings / 市場 / 對話 | 多源，尚未統一流水線 |
| **處理層** | domains/book-digest 等採集器，產出結構化卡片 | 部分已建，待補全 |
| **策展層** | domains/curate，跨源聚合 + 去重 + 主題化 | 框架存在，待完善 |
| **決策層** | 三個域：trading-discipline / product-experience / personal-capability | 投資完整，另兩域缺反饋表 |
| **反饋層** | 投資已有「交易日志 × 規則歸因」閉環 | 另兩域待複製此模式 |

---

## 6. 範圍邊界

**In scope**：採集 → 處理 → 策展 → 決策 → 反饋的完整閉環，覆蓋三個決策域。

**Out of scope**：多用戶協作、實時推送通知、外部 API 服務化、知識完備性追求。

---

## 7. 成功指標

初始基線為估算值，待系統跑通後以實測數據校準。

| 指標 | 定義 | 初始目標值 |
|------|------|-----------|
| **採集效率** | 任意輸入源 → 結構化卡片的處理時間 | ≤ **15 分鐘**（X） |
| **決策可追溯性** | 重要決策中能追溯到卡片來源的比例 | ≥ **80%**（Y） |
| **反饋有效性** | 每 **3 個月**（N），根據反饋優化的規則比例 | ≥ **20%**（Z） |

---

## 8. 非目標

- 不重建現有評估體系（trading-discipline 模式有效，複製即可）
- 不追求完全自動化（人在環路是有意設計，保留人工判斷節點）
- 不追求知識完備性（只覆蓋影響重要決策的領域，避免收藏癖）
- 不追求多人協作（單用戶系統，複雜度控制在個人可維護範圍內）
