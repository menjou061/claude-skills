# Obsidian Inbox 笔记 Skill

**版本：v0.1.0**

---

## 配置（集中修改区）

修改此处即可全局生效，无需动工作流其他部分：

```
INBOX_PATH  = /Users/menjou/Library/Mobile Documents/iCloud~md~obsidian/Documents/menjou-zero/00-Inbox
FILENAME    = YYYY-MM-DD-<slug>.md   （slug: 小写英文 + 连字符，不含中文/空格）
DEFAULT_TAG = inbox
MAX_TAGS    = 4（含 inbox）
```

---

## 触发条件

当用户消息中包含以下任一意图时，**必须**使用此 Skill：

- "记备忘" / "记一下" / "存成笔记" / "存到 Obsidian"
- "帮我整理今天的对话" / "整理要点"
- "存到收件箱" / "写到 inbox"
- 带有定时意图："晚上/明天写到 Obsidian"

---

## 工作流程（严格按顺序执行）

### 步骤 1：理解需求

确认以下信息（如用户已明确，跳过询问）：

- 笔记主题是什么（对话核心话题）
- 是否需要行动项（有则生成 `- [ ]` 列表）
- 是否有外部链接需要记录

如果需求非常明确，可直接进入步骤 2。

### 步骤 2：阅读参考资料

1. **必读**：[`references/note-template.md`](./references/note-template.md) — 标准笔记结构，严格按此模板输出

### 步骤 3：生成文件名

- 从对话主题推断标题（5 词以内，中文或英文均可）
- 生成 slug：标题小写 + 空格转连字符 + 去掉非 `a-z0-9-` 字符
- 文件名格式：`YYYY-MM-DD-<slug>.md`
- 示例：`2026-05-11-financial-services-research.md`

### 步骤 4：生成笔记内容

严格按 `references/note-template.md` 的结构填写，规则：

- 从对话提炼要点，**不得**照搬对话原文
- 行动项用 `- [ ]`，便于 Obsidian Tasks 插件跟踪
- 有 URL 记入参考链接节；无则省略该节
- 涉及敏感信息（API key、密码等）**不得**写入文件

### 步骤 5：写入文件

将完整笔记写入：

```
<INBOX_PATH>/<filename>.md
```

### 步骤 6：自检（写入后必做）

对照下方"强制规则清单"逐条检查，有违反立即修正后重写。

---

## 强制规则清单（必须全部满足）

### 文件名

- [ ] 文件名**只含** `a-z 0-9 -` 和 `.md`，不含中文、空格、下划线
- [ ] 文件名以当天日期 `YYYY-MM-DD-` 开头

### Frontmatter

- [ ] `date` 字段格式为 `YYYY-MM-DD`
- [ ] `tags` 列表**必须**包含 `inbox`，其余 topic tag 不超过 3 个
- [ ] 有明确来源 URL 时包含 `source` 字段，否则省略

### 正文结构

- [ ] 严格遵循 `note-template.md` 的节顺序，不得随意增删必填节
- [ ] `## 核心要点` 至少 3 条，每条独立成行
- [ ] 行动项全部使用 `- [ ]` 格式，**禁止**用 `-`、`*` 或数字列表替代
- [ ] 正文**不得**复述行动项的来龙去脉，保持简洁

### 安全

- [ ] 不包含任何 API key、token、密码、私钥等敏感字符串

---

## 输出格式

写入完成后，向用户输出：

1. **文件路径**（完整绝对路径）
2. **笔记概要**：字数 + 行动项数量
3. **标签列表**：写入了哪些 tags

---

## 不确定时先问

遇到以下情况**必须先询问**：

- 对话内容跨多个主题，不确定用哪个作为主标题
- 用户提到的内容含可能敏感的信息，不确定是否需要脱敏
- 行动项的负责人或截止日期不明确，但用户明显希望记录

---

## 适用边界（哪些情况不用本 Skill）

- 用户要直接编辑已有 Obsidian 笔记 → 不用本 Skill，按用户要求直接编辑文件
- 用户要生成复杂报告/文档 → 本 Skill 只生成收件箱速记，不适合长文档
- 用户要定期自动归档 → 本 Skill 是单次手动触发，自动化需配合 cron 另外实现
