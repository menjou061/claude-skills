# 笔记模板（note-template）

**用途**：`obsidian-inbox` Skill 生成笔记时严格按此结构输出。
**维护**：修改此文件即可更新所有新笔记的结构，无需改动 SKILL.md。

---

## 模板结构

```markdown
---
title: <笔记标题>
date: YYYY-MM-DD
tags:
  - inbox
  - <topic-tag-1>
  - <topic-tag-2>
source: <来源 URL，无则删除此行>
---

# <笔记标题>

## 核心要点

- <要点 1>
- <要点 2>
- <要点 3>
<!-- 3 条以上，每条独立成行，提炼自对话，不照搬原文 -->

## 背景

<1-2 句话说明这条笔记的来龙去脉，为什么值得记录>

## 行动项

- [ ] <下一步 1>
- [ ] <下一步 2>
<!-- 用 - [ ] 格式，便于 Obsidian Tasks 插件跟踪；无行动项则删除此节 -->

## 参考链接

- [<标题>](<URL>)
<!-- 有 URL 才保留此节；无则整节删除 -->
```

---

## 各节说明

| 节 | 是否必填 | 说明 |
|---|---|---|
| frontmatter | 必填 | date + tags(含 inbox) 必须有；source 有 URL 才加 |
| `## 核心要点` | 必填 | 至少 3 条，提炼要点 |
| `## 背景` | 必填 | 1-2 句，说明为什么记这条笔记 |
| `## 行动项` | 有则填 | 无明确下一步可删除整节 |
| `## 参考链接` | 有则填 | 无 URL 可删除整节 |

---

## 扩展节（v0.2+ 规划，暂不使用）

以下节在未来版本中可能加入，目前**不得**在笔记中生成：

- `## 关联笔记` — 与 vault 内其他笔记的双向链接
- `## 原始对话摘要` — 保留对话片段供回溯
